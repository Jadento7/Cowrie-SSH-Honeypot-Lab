# Honeypot Labs

## Overview

This repository documents a series of beginner-friendly honeypot labs focused on deploying, testing, and analyzing Cowrie SSH honeypot activity in a controlled local environment.

The labs start with deploying a basic Cowrie SSH honeypot and then move into analyzing the logs generated by controlled SSH login attempts. The goal is to practice defensive security concepts such as deception technology, log collection, event review, command analysis, and honest documentation of findings.

These labs are designed for learning and portfolio documentation. They should not be treated as production honeypot deployments or real-world threat intelligence unless deployed safely in a properly isolated internet-facing environment.

## Key Concepts Practiced

### Honeypot Deployment

The first lab shows how to deploy Cowrie as a fake SSH service. Cowrie intentionally simulates a vulnerable SSH server so login attempts and fake shell activity can be captured safely.

### Controlled Attack Simulation

The SSH login attempts in these labs were generated manually in a local environment. This makes the activity safe, repeatable, and appropriate for beginner documentation.

### Log Collection

Cowrie logs were collected using Docker logs:

```bash
docker logs cowrie > cowrie-session.log
```

This created a log file that could be searched and analyzed later.

### Log Analysis

The second lab uses basic Linux commands to extract useful data from the Cowrie logs:

```bash
grep -i "login attempt" logs/cowrie-session.log
grep -i "New connection" logs/cowrie-session.log
grep "CMD:" logs/cowrie-session.log
```

The extracted data was saved into smaller files such as:

```text
login-attempts.txt
source-connections.txt
commands.txt
```

### Honest Interpretation

The observed source IP was local Docker networking, not a real external attacker. The README avoids exaggerating the results and clearly explains that the activity was from a controlled local test.

## How to Use This Repository

Complete the labs in order:

```text
01-cowrie-ssh-honeypot
02-cowrie-log-analysis
```

Lab 01 should be completed first because Lab 02 depends on Cowrie logs generated from the honeypot deployment.

## What I Learned

Through these labs, I learned how to deploy a basic SSH honeypot and analyze the activity it captures. I practiced using Docker to run Cowrie, testing SSH access against a fake service, exporting logs, filtering events, and summarizing the results.

The biggest lesson from this project is that deploying a security tool is only part of the work. The real value comes from reviewing the data, understanding what happened, and documenting the findings accurately.

I also learned the importance of not exaggerating lab results. Since this was a local test, the captured source IP represents Docker networking, not a real internet attacker.

## Security and Ethics Notice

These labs were created for educational use in a controlled local environment.

Do not expose honeypots to the public internet unless you understand isolation, firewalling, logging, and risk. Do not store real passwords, SSH keys, personal files, or sensitive information on a honeypot.

Do not retaliate against source IPs, scan attackers, or perform unauthorized activity. Honeypots are for observation, learning, and defensive analysis only.
